# 場景 7：Agent 文檔大師 (80% Agent)

## 🎯 學習目標
- 體驗 **Agent 80% 模式**：從架構師升級為文檔專家
- 讓 Agent 自主理解程式碼並生成完整技術文檔體系
- 學習如何讓 Agent 創建多種類型的開發者文檔
- 掌握 Agent 驅動的文檔自動化流程

## 📝 場景說明
您有一個功能完整的**任務管理 API 系統**，但完全缺乏文檔。與傳統手動撰寫文檔不同，您將學習如何讓 **Agent 作為技術文檔專家**，從深度理解程式碼架構開始，自主生成 API 文檔、開發指南、程式碼註解、部署說明等完整文檔體系。

## 🛠️ 專案結構
```
07-documentation/
├── README.md (本檔案)
├── index.html (文檔實驗室界面)
├── style.css (精美的展示界面)
├── api-demo.js (API 演示邏輯)
├── api-server/ (待文檔化的 API 系統)
│   ├── server.js (Express 主服務器)
│   ├── routes/ (API 路由)
│   ├── models/ (數據模型)
│   ├── middleware/ (中間件)
│   └── utils/ (工具函數)
└── docs/ (Agent 生成的文檔目錄)
```

## 📋 文檔挑戰

### 缺失的文檔類型
1. **📚 API 參考文檔**：詳細的端點說明、參數、回應格式
2. **👨‍💻 開發者指南**：快速開始、使用範例、最佳實踐
3. **💬 程式碼註解**：JSDoc 註解、內聯說明
4. **🚀 部署文檔**：環境配置、部署步驟、故障排除
5. **🧪 測試文檔**：測試指南、範例測試用例
6. **🏗️ 架構說明**：系統設計、數據流、技術決策

### API 系統概覽
- **用戶管理**：註冊、登入、個人資料
- **任務管理**：CRUD 操作、狀態管理、優先級
- **項目管理**：項目創建、任務關聯、團隊協作
- **認證系統**：JWT 令牌、權限控制
- **數據模型**：用戶、任務、項目關聯

## 📚 Demo 劇本

### 🌟 **重點：Agent 80% 體驗**
Agent 成為**技術文檔專家**，不只生成文檔，還要自主理解系統並創建用戶友好的文檔體驗。

### 階段 1：Agent 系統理解 (3分鐘)

#### 步驟 1：讓 Agent 深度分析 API 系統
```
@workspace 請深度分析這個任務管理 API 系統，包括：
1. 理解整體架構和數據流
2. 識別所有 API 端點和功能
3. 分析數據模型和關聯關係
4. 評估系統的複雜度和特色
5. 確定文檔化的重點和挑戰
```

**⚡ 觀察重點**：
- Agent 會自主探索整個程式碼庫
- 理解業務邏輯和技術架構
- 識別文檔化的關鍵需求

#### 步驟 2：讓 Agent 設計文檔架構
```
基於你的分析，請設計一個完整的文檔體系架構：
1. 文檔類型和優先級
2. 目標受眾分析（開發者、運維、產品）
3. 文檔結構和導航設計
4. 內容策略和撰寫風格
5. 維護和更新機制
並說明為什麼這樣設計。
```

### 階段 2：Agent 主導文檔生成 (10分鐘)

#### 步驟 3：讓 Agent 生成 API 參考文檔
```
請生成完整的 API 參考文檔，包括：
- 所有端點的詳細說明
- 請求/回應格式和範例
- 錯誤碼和處理方式
- 認證和權限說明
- 使用 OpenAPI/Swagger 格式
確保文檔準確且易於理解
```

#### 步驟 4：讓 Agent 創建開發者指南
```
請創建完整的開發者指南：
- 快速開始教學
- 常見使用場景和範例
- SDK 和工具推薦
- 最佳實踐和注意事項
- 故障排除指南
讓新開發者能快速上手
```

#### 步驟 5：讓 Agent 增強程式碼註解
```
為所有程式碼添加詳細的註解：
- JSDoc 格式的函數註解
- 複雜邏輯的內聯說明
- 數據模型的屬性說明
- 中間件和工具的使用說明
確保程式碼自說明且易維護
```

#### 步驟 6：讓 Agent 生成部署和架構文檔
```
創建系統部署和架構文檔：
- 環境需求和配置指南
- Docker 容器化說明
- 雲端部署步驟
- 系統架構圖和說明
- 效能優化建議
```

### 階段 3：Agent 文檔優化 (2分鐘)

#### 步驟 7：讓 Agent 建立互動式文檔
```
請建立互動式文檔網站：
1. 整合 API 測試功能
2. 提供即時程式碼範例
3. 建立搜尋和導航功能
4. 添加用戶回饋機制
5. 確保響應式設計
並在瀏覽器中展示成果
```

## 💡 重點技巧

### 🤖 Agent 80% 模式要點：
1. **深度理解**：Agent 能理解複雜的系統架構和業務邏輯
2. **創意表達**：Agent 能用清晰易懂的方式表達技術概念
3. **用戶導向**：Agent 考慮不同受眾的需求和使用場景
4. **品質保證**：Agent 主動確保文檔的準確性和完整性

### 📋 有效的 Agent 指令模式：
- "深度分析系統並設計文檔架構"（而非"寫一些文檔"）
- "創建完整的開發者體驗"（而非"生成 API 說明"）
- "建立互動式文檔網站"（而非"寫 README 檔案"）
- "確保文檔的長期維護性"（而非"檢查拼字錯誤"）

### 🚫 避免的做法：
- 不要限制 Agent 的創意和表達方式
- 不要只要求單一類型的文檔
- 不要忽略文檔的用戶體驗設計
- 不要忘記讓 Agent 驗證文檔品質
